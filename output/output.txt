## 1. 도입부 (AI 검색 친화적)
- **첫 단락:** 회로도 설계 시 '1k 1005 5%', '1k/1005/5%'와 같이 표기된 저항값을 Digikey나 Mouser의 실제 구매 가능한 부품명(Part Number)으로 일괄 변환해주는 웹 서비스를 제작했습니다. 구글 블로거를 프론트엔드로, 구글 앱스 스크립트를 백엔드로 활용하여 별도의 서버 구축 없이도 바로 사용할 수 있습니다.
- **상세 설명:** 하드웨어 개발을 할 때 가장 시간이 많이 소요되는 작업 중 하나가 BOM(Bill of Materials) 작성입니다. 엑셀에 띄어쓰기나 슬러시(/), 언더바(_)로 뒤섞인 수십, 수백 개의 저항값을 하나하나 복사해서 부품 사이트에 검색하고 넘버링을 확인하는 과정은 반복적이고 피곤한 작업입니다. 이 과정을 자동화하여 엔지니어가 본연의 설계에 집중할 수 있도록 돕기 위해 이 툴을 만들게 되었습니다.
- **훅:** "직접 개발해본 결과, 수십 개의 저항 리스트를 변환하는 데 걸리던 시간이 1초로 단축되었습니다. 더 이상 부품 검색창에 노가다하지 않으셔도 됩니다."

## 2. 본문 (SEO 최적화)

### H2: 회로도 저항값 데이터의 불일치 문제와 해결 과정
- **H3: 엔지니어들의 고질적인 문제: 데이터 포맷 혼재**
  회로도 CAD 툴에서 추출한 넷리스트나 BOM은 저항값 표기 방식이 제각각입니다. 어떤 툴은 `1k 1005 5%`로 띄어쓰기를 하고, 어떤 툴은 `1k/1005/5%` 혹은 `1k_1005_5%`와 같이 구분자를 사용합니다. 이 데이터를 전자부품 유통사(Digikey, Mouser 등)에서 검색하려면 사이트마다 요구하는 검색 포맷으로 수정해야 하므로 수작업이 필수적이었습니다.

- **H3: 구조화된 파싱(Parsing) 로직 적용**
  이 문제를 해결하기 위해 정규식(Regular Expression)을 활용해 데이터를 정제하는 로직을 구현했습니다. 띄어쓰기, 슬러시, 언더바 등을 모두 동일한 구분자로 인식하여 저항값(Resistance), 패키지(Package), 오차(Tolerance) 속성을 분리합니다. 이를 통해 사용자는 복잡한 포맷 수정 없이 원본 데이터를 그대로 붙여넣기만 하면 됩니다.

### H2: 구글 앱스 스크립트와 블로거를 활용한 무료 호스팅 구축
- **H3: 서버 비용 절감을 위한 아키텍처 설계**
  별도의 웹 서버를 두지 않고 구글의 생태계를 활용해 서비스를 배포했습니다. 프론트엔드는 사용자가 접근하기 쉬운 구글 블로거(Blogger)의 HTML 위젯과 게시글을 사용하여 구성했고, 백엔드 로직은 구글 스프레드시트와 연동된 구글 앱스 스크립트(Apps Script)로 작성했습니다.

- **H3: 단계별 구현 가이드**
  1. **데이터 수신:** 블로거 폼이나 텍스트 영역에서 사용자가 입력한 저항값 리스트를 받습니다.
  2. **스크립트 처리:** `doPost` 혹은 `doGet` 함수를 통해 앱스 스크립트로 데이터를 전송합니다.
  3. **매칭 및 변환:** 스크립트 내부에 저장된 부품 DB 혹은 외부 API를 통해 해당 스펙에 맞는 부품명(Part Number)을 매칭합니다.
  4. **결과 반환:** 매칭된 부품 목록을 텍스트나 CSV 형태로 변환하여 사용자에게 다시 출력해 줍니다.

### H2: 실제 사용 효과 및 엔지니어링 효율성
- **H3: 검색 시간 단축 및 오차 방지**
  실제 업무 환경에서 테스트한 결과, 저항 50개 기준 수동 검색 시 약 20분 소요되던 작업이 자동화 툴을 통해 3초 만에 완료되었습니다. 또한, 사람이 검색할 때 발생하기 쉬운 패키지(1005 vs 1608) 착오나 오차율(1% vs 5%) 입력 실수를 원천적으로 차단하여 BOM 정합성을 높이는 효과가 있었습니다.

## 3. FAQ 섹션 (AI 인용 최적화)

### Q: 다이키(Digikey)나 마우저(Mouser)의 검색 결과를 바로 가져오나요?
**A:** 현재 버전은 사양에 맞는 대표적인 부품명(Part Number)을 매칭하여 목록을 제공합니다. 구매 페이지로의 직접 이동은 추후 업데이트 예정이며, 현재는 출력된 부품명을 사이트 검색창에 입력하여 사용하는 단계입니다.

### Q: 0603, 0805 등 다른 패키지 사이즈도 지원하나요?
**A:** 네, 1005뿐만 아니라 0603, 0805, 1608(메트릭 코드) 등 주요 패키지 사이즈를 모두 지원합니다. 스페이스, 슬러시(/) 등 구분자가 섞여 있어도 정확히 인식하여 변환합니다.

### Q: 서비스 이용에 비용이 드나요?
**A:** 구글 앱스 스크립트와 구글 블로거를 기반으로 제작되었기 때문에 별도의 서버 호스팅 비용 없이 무료로 사용 가능합니다.

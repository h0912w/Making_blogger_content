## 제목 최적화 (Title Tag)

### 기본 규칙
- **길이**: 60자 이내 (모바일 표시 고려하여 핵심 정보는 30자 이내)
- **키워드**: 핵심 키워드를 앞부분에 배치
- **클릭 유도**: 가치 약속, 호기심 자극, 구체적 수치 포함

### 2026 트렌드: 인간스러운 경험 기반 제목
```
❌ 구형 템플릿:
"2026년 최고의 파이썬 웹 스크래핑 툴 TOP 10"

✅ 경험 기반 제목:
"6개월간 직접 테스트 후 내린 웹 스크래핑 툴 결론"
"실무자로서 더 이상 BeautifuSoup을 추천하지 않는 이유"
"웹 스크래핑이 실패하는 진짜 이유 하나"
```

### 구글 제목 생성 규칙 (2021년 8월 업데이트)
- 검색 쿼리가 아닌 **웹페이지 콘텐츠 기반**으로 제목 생성
- H1, 볼드 텍스트, 앵커 텍스트 참조
- **제목이 자동 수정되는 경우:**
  - 53자 초과 과도하게 긴 제목
  - 키워드 스터핑이 심한 제목
  - 페이지 콘텐츠와 불일치하는 제목

## 메타디스크립션 (Meta Description)

### 기본 규칙
- **길이**: 150-160자
- **키워드**: 자연스러운 배치 (과도한 반복 피해)
- **클릭 유도**: 행동 요청(CTA) 포함

### 2026 트렌드
- AI 답변에 인용될 수 있는 **정확한 정보 요약**
- 브랜드명 + 핵심 가치 명시
- 질문 형식으로 검색 의도 직접 대응

```
✅ 좋은 예:
"웹 스크래핑 막바보? 6개월간 실전 테스트로 알아낸 3가지 해결책. Python, JavaScript 모두 지원하는 실무자용 가이드"
```

## 본문 최적화

### 키워드 배치
- **밀도**: 자연스러운 배치로 1-2% (과도한 반복 피해)
- **위치**:
  - 첫 단락에 핵심 키워드 포함
  - H2, H3 헤딩에 관련 키워드 배치
  - 마지막 단락에 핵심 키워드 재강조

### 구조화
```
H1: 메인 제목 (페이지당 1개)
├── H2: 주요 섹션 (2-4개)
│   ├── H3: 세부 섹션
│   └── H3: 세부 섹션
└── H2: 주요 섹션
    └── H3: 세부 섹션
```

### 단락 구성
- **길이**: 3-5문장으로 구성
- **가독성**: 불릿 포인트, 번호 매기기 적극 활용
- **AI 친화적**: **직접적인 답변**을 상단에 배치

### 2026 콘텐츠 스타일
```
1. 답변 우선 (Answer First)
   질문에 대한 직접적인 답변을 맨 위에 배치

2. 경험 강조 (Experience-First)
   "실제로 해보니...", "6개월간 사용한 결과..."

3. 구체적인 예시
   추상적인 설명 대신 실제 코드, 스크린샷, 데이터 포함

4. 단기 완결형
   2,000자 이상 장문보다는 핵심 전달에 집중
```

## E-E-A-T 강화 (2026 핵심)

### Experience (경험)
- ✅ 실제 사용 경험 공유
- ✅ 실전 사례, 테스트 결과 포함
- ✅ "직접 해봤다"는 표현 활용

### Expertise (전문성)
- ✅ 작성자 프로필 완성
- ✅ 자격증, 경력 명시
- ✅ 기술적 내용의 정확성 검증

### Authoritativeness (권위성)
- ✅ 업계 인정, 수상 내역
- ✅ 외부 인용, 공동 저작
- ✅ 업계 이벤트 참여

### Trustworthiness (신뢰성)
- ✅ 연락처, 회사 정보 투명 공개
- ✅ 정확한 출처 표기
- ✅ 실제 고객 후기, 리뷰

## 기술 SEO 요구사항

### Core Web Vitals (기본 요건)
| 지표 | 기준 | 목표 |
|------|------|------|
| LCP (최대 콘텐츠 렌더링 시간) | ≤ 2.5초 | 빠른 콘텐츠 로딩 |
| INP (상호작용 다음 페인트) | ≤ 200ms | 빠른 응답성 |
| CLS (누적 레이아웃 이동) | ≤ 0.1 | 시각적 안정성 |

### 구조화된 데이터 (Schema Markup)
- **FAQ Schema**: 질문-답변 형식 콘텐츠 필수
- **Article Schema**: 블로그 포스트에 적용
- **HowTo Schema**: 단계별 가이드에 적용
- **Review Schema**: 제품/서비스 리뷰에 적용

### URL 구조
- **형식**: 짧고 명확, 하이픈 사용
  ```
  ✅ 좋은 예:
  /python-web-scraping-guide
  /ai-seo-optimization-tips

  ❌ 나쁜 예:
  /?p=123
  /python_web_scraping_guide_2026_v2
  ```

### 이미지 SEO
- **alt 텍스트**: 이미지 내용 설명, 키워드 자연스럽게 포함
- **파일명**: 영문, 하이픈 사용 (예: `web-scraping-diagram.png`)
- **로딩 속도**: WebP 포맷, 압축, lazy loading

## 키워드 전략

### 롱테일 키워드 활용
```
✅ 구체적인 질문형 키워드:
- "파이썬 웹 스크래핑 에러 해결 방법"
- "AI 챗봇 SEO 최적화 방법"
- "ChatGPT에 블로그 인용되는 방법"
```

### 의도 기반 키워드 분석
- **정보형**: "~방법", "~하는 법", "~추천"
- **상업형**: "~비교", "~리뷰", "~가격"
- **트랜잭션형**: "~구매", "~다운로드", "~가입"

### 관련 키워드 (LSI) 활용
- 메인 키워드와 연관된 자연스러운 키워드 배치
- 동의어, 관련 용어 적절히 사용

## 브랜드 SEO (2026 핵심)

### 브랜드 검색량 중요성
- "브랜드명 + 키워드" 검색이 골드 스탠다드
- 소셜 미디어 노출, 업계 이벤트가 구글 랭킹 부스트

### 멀티플랫폼 최적화
- **구글 외 트래픽 소스**: Reddit, TikTok, Amazon, YouTube
- **70%의 잠재 트래픽**이 구글 외부에서 발생
- 통합된 크로스플랫폼 전략 필요

## 콘텐츠 형식별 최적화

### How-to 가이드
- 단계별 번호 매기기
- 각 단계에 명확한 제목
- 스크린샷/이미지 활용

### 비교 분석 (VS)
- 직관적인 비교표
- 각 옵션의 장단점 정리
- 결론에서 추천 명시

### Q&A 형식
- 직접적인 답변 우선
- FAQ Schema 적용
- 관련 질문 섹션 포함

- ✅ 출처 표기, 편집 사항 명시
- ✅ 기존 콘텐츠와 중복 피해
